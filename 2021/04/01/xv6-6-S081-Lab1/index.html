pingpong

Pingpong题目

Write a program that uses UNIX system calls to ''ping-pong'' a byte between two processes over a pair of pipes, one for each direction. The parent should send a byte to the child; the child should print ": received ping", where is its process ID, write the byte on the pipe to the parent, and exit; the parent should read the byte from the child, print ": received pong", and exit. Your solution should be in the file user/pingpong.c.

分析

这道题没什么特别难的知识点和算法，就是要熟悉一下fork()的使用以及对pipe()的使用，需要注意

  1. fork()函数: fork()函数在执行后会在内核中复制一个与当前进程几乎完全一样的进程,并进行进程的调度

    • 对父进程fork()返回值为子进程的PID(ProcessID)
    • 对子进程fork()返回值为0

    据此我们可以判断当前进程是父进程还是子进程

  2. pipe管道: 我对操作系统的学习才刚刚开始,对管道的理解并不深刻,单纯讲讲管道在这个lab里怎么用.

    首先,需要创建(不需要赋初值)的int型数组(大小为2) 然后,以刚刚的数组作为参数调用pipe(),系统会创建管道 最后再调用fork(),数组中会存储两个进程的File descriptor,管道建立完成!

    在具体的使用中,管道的第一个元素作为该进程中管道的读取端,第二个元素作为管道的输入端.

Primes(仔细说说)

Primes题目

Write a concurrent version of prime sieve using pipes. This idea is due to Doug McIlroy, inventor of Unix pipes. The picture halfway down this page and the surrounding text explain how to do it. Your solution should be in the file user/primes.c.

菜鸡随便翻译一下

用管道写一个多进程版的质数生成,这个点子来自于Doug Mcllroy, Unix Pipe的发明者. 下面这个网页的图片和它周围的文字解释了怎么做这件事,你的解决方案应当保存在 user/primes.c文件里.

分析

首先,看到链接我就先点进去瞅一眼这是个什么东西。哦,是讲CSP Threads的,题目用了这篇文章的样例作为要求,大概讲了实现所使用的算法 大致的流程图以及伪代码

p = get a number from left neighbor
print p
loop:
    n = get a number from left neighbor
    if (p does not divide n)
        send n to right neighbor

这一段是进程间通过管道传输并且判断是否为质数的一个大致思路，那么我们现在就该考虑如何实现这一问题 我们要注意到：

  • 第一个进程与其他子进程不同，它并没有从父进程获取 n 的这个过程，而是直接遍历，所以我们应当判断fork()的返回值是不是0
  • 当到达35后，所有的进程将自下而上(或者说自右向左的)关闭
  • 文件描述符的资源有限，也就是说，不用的要记得关。那么什么叫不用的呢？很简单，管道不用往左写，也不用往右读，这就省出来了。

find

题目

Write a simple version of the UNIX find program: find all the files in a directory tree whose name matched a string. Your solution should be in the file user/find.c.

分析

一些必须看的东西

在真的开始写这道题之前，xv6Book的第一章真的不看不行了，之前还可以稍微摸会儿鱼（其实本来就应该早点看的），这道题不看xv6Book对ls.c根本无法理解，这里我就随便说几个这道题要用到的特别特别重要的点，剩下的大家自己去看

  1. 同一个实际文件(这里被称作是inode)可以对应好几个存在于相同或者不同目录下的入口(类比Windows下的Shortcut）
  2. 系统调用fstat(int fd, struct stat*)可以查看一个inode的信息，包括文件大小、文件类型、指向该inode的link的数量等等
  3. 文件有三种：目录(T_DIR)、文件(T_FILE)和设备(T_DEVICE)
  4. 对一个目录，有一个dirent存储其信息，里面有目录的名字和inode数，如果这个inode数(inum)等于0的话，这个目录就是一个空目录

子曰：有现成的不用是混蛋

我们在提示中已经看到题目提醒我们要参考user/ls.c的内容了，那么不用就太可惜了，这就Ctrl+C&Ctrl+V 但是需要注意，不造轮子虽然是美德，但当懒狗当的太彻底一个字都不改那就有问题了，仔细一看ls.c有很多需要修改的地方:

  1. fmtname()应该是用来格式化路径的，但我们这道题并不需要
  2. ls.c所找到的文件和路径都需要在标准输出里输出，但我们的find不需要
  3. 当ls.c里读到一个路径下面还是路径时，他一点都不在意的，而我们需要递归查询
  4. ....

接下来大家整理一下思绪，把程序实现就行了
